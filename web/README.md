# Bake-off web app (Vercel)

The read + live-price layer for the paper-trading bake-off. Static frontend + one serverless
function. The bake-off **state is produced locally** by the daily tick and published here as
`state.json`; this site never runs the agents.

## How it works
- `index.html` / `styles.css` / `app.js` — the dashboard. On load it fetches `state.json`
  (the bake-off), then polls `/api/quotes` every 15s for **live prices**.
- `api/quotes.js` — serverless function; fetches quotes from Finnhub using `FINNHUB_API_KEY`
  (a Vercel env var), CDN-cached 15s.
- `state.json` — committed snapshot of the bake-off, regenerated by `tools/build_dashboard.py`
  (run from the repo root after each tick). A `git push` redeploys the site with fresh state.

## Local dev
```
cd web
vercel dev          # serves static + /api/quotes (reads web/.env for FINNHUB_API_KEY)
```
Or static-only (no live quotes): `python3 -m http.server 8000` then open http://localhost:8000.
For `vercel dev`, put `FINNHUB_API_KEY=...` in `web/.env` (gitignored).

## Deploy
1. `vercel link` (or import the repo in the Vercel dashboard) with **Root Directory = `web`**.
2. Add env var **`FINNHUB_API_KEY`** in Vercel → Project → Settings → Environment Variables.
3. `vercel --prod` (or push to the linked branch for auto-deploy).

The daily local tick → `git push` → Vercel auto-deploys the new `state.json`. Live prices update
on their own in the browser; only bake-off state needs a push.
