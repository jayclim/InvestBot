# InvestBot web app (Next.js + Vercel)

The read + live-price layer for the bake-off — a **Next.js** (App Router) app. It fetches the
bake-off state and live-polls real prices. The bake-off **state is produced locally** by the daily
tick; this site never runs the agents.

## How it works

- `app/page.jsx` + `components/*` — the dashboard. On load it fetches `public/state.json`
  (the bake-off), then polls `/api/quotes` every 15s for **live prices**.
- `app/api/quotes/route.js` — serverless route; fetches quotes from Finnhub using
  `FINNHUB_API_KEY` (an env var), CDN-cached 15s so many viewers → few provider calls.
  Sibling routes: `/api/news` (Finnhub headlines) and `/api/intraday` (pre/after-hours candles, Yahoo).
- `public/state.json` — committed snapshot of the bake-off (incl. the S&P 500 benchmark series),
  regenerated by `tools/build_dashboard.py` (run from the repo root after each tick). A `git push`
  redeploys. Siblings published the same way: `public/history.json` (per-symbol close history for the
  charts) and `public/news.json` (the daily headline cache — a free static asset with links).
- **Display scale:** the real paper books are $100, but every dollar/share figure on the site is shown
  scaled to a **$10,000 notional** for readability — applied once in `tools/build_dashboard.py`
  (`DISPLAY_SCALE`); per-share prices and percentages are unscaled. Ticker → company name lives in
  `lib/names.js`.

## Local dev

```bash
cd web
npm install
npm run dev          # http://localhost:3000   (reads web/.env.local for FINNHUB_API_KEY)
```

Put `FINNHUB_API_KEY=...` in `web/.env.local` (gitignored). Without it the dashboard still loads
from `state.json` — only the live-price panel is disabled.

## Deploy to Vercel

**CLI (one-shot live URL):**

```bash
cd web
vercel                          # first run links/creates the project; accept the Next.js defaults
vercel env add FINNHUB_API_KEY  # paste the key; choose Production + Preview + Development
vercel --prod
```

Run it from **inside `web/`** so the Next.js app is the project root.

**Git integration (auto-deploy on push):** import the repo in the Vercel dashboard with
**Root Directory = `web`**, add the `FINNHUB_API_KEY` env var, and deploy. After that, each
`git push` redeploys — the daily tick writes `public/state.json` → push → fresh state on the site.
Live prices update on their own via `/api/quotes`; only the bake-off state needs a push.
